# Versioning Policy

This project uses Semantic Versioning.

## Pattern

- `MAJOR.MINOR.PATCH`

## Meaning

- `PATCH`
  Use for bug fixes, layout corrections, small documentation updates, and non-breaking cleanup.
- `MINOR`
  Use for new user-facing features that remain backward compatible.
- `MAJOR`
  Use for breaking changes, major architecture shifts, or behavior changes that significantly alter how users interact with the tool.

## Current Version Track

- `1.0.0`
  Initial public release with core rule management.
- `1.1.0`
  Bilingual UI, metadata, and repository publishing setup.
- `1.2.0`
  First-run wizard, clearer status colors, details view, toast notifications, and release automation.

## Suggested Roadmap

- `1.3.0`
  Custom presets, editable rule targets, and persisted advanced settings.
- `1.4.0`
  Logging, diagnostics, and import/export of rule configurations.
- `2.0.0`
  Multi-rule management, richer profiles, and a larger application structure split into a more complete desktop utility.
